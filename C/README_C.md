# ColorLight LED Text Controller (C++ Version) 상세 설명서

이 폴더는 기존 파이썬 기반의 LED 컨트롤러를 고성능 C++로 포팅한 버전입니다. 라즈베리파이와 같은 임베디드 환경에서 최소한의 리소스로 동작하며, 외부 키보드를 통한 실시간 제어에 최적화되어 있습니다.

## 📂 파일 구성 및 역할

### 1. 메인 엔진 (Core Engine)
*   **`main.cpp`**: 프로그램의 중심점입니다. 설정을 로드하고, 사용자 입력을 받아 렌더러와 컨트롤러를 연결하는 전체 흐름을 제어합니다.
*   **`ColorLightController.hpp / .cpp`**: 네트워크 통신 담당. `scapy` 라이브러리 없이 리눅스 Raw Socket을 직접 제어하여 LED 패널에 이더넷 패킷을 초고속으로 전송합니다.
*   **`TextRenderer.hpp / .cpp`**: 그래픽 렌더링 담당. OpenCV나 SDL 같은 무거운 라이브러리 없이 `stb_truetype.h`를 사용하여 폰트 파일에서 직접 픽셀 데이터를 추출하고 상하좌우 정중앙에 정렬합니다.

### 2. 리소스 및 설정 (Resources & Config)
*   **`LED_Config`**: 패널 해상도, 배경색, 글자색, 네트워크 인터페이스 이름 등 모든 설정이 담긴 JSON 형식의 파일입니다.
*   **`font/`**: 사용되는 폰트 파일(`GothicBold.ttf`)이 저장된 폴더입니다.
*   **`stb_truetype.h`**: 폰트 렌더링을 위한 경량 헤더 라이브러리입니다.

### 3. 빌드 및 배포 (Build & Deploy)
*   **`Makefile`**: 복잡한 컴파일 과정을 `make` 명령 하나로 압축해 줍니다.
*   **`setup_autostart.sh`**: 라즈베리파이 전용 자동 실행 설정 스크립트입니다. `.bashrc`와 연동되어 부팅 후 자동 로그인 시 즉시 키패드 입력을 받을 수 있는 상태로 만들어줍니다.

## 🛠 주요 기술 사양

### 1. 인터렉티브 키패드 지원 (NumLock Off 대응)
일반적인 `getline` 방식이 아닌 터미널 **Raw Mode** 제어를 구현했습니다. 이를 통해 NumLock이 꺼진 상태에서 발생하는 ANSI 이스케이프 시퀀스(예: `^[[A`)를 가로채서 실제 숫자(`8`)로 매핑합니다. 사용자는 키패드 상태와 상관없이 항상 숫자를 입력할 수 있습니다.

### 2. 정밀한 세로 중앙 정렬
글자의 종류(g, y처럼 아래로 뻗는 글자 등)와 상관없이 실제 그려지는 '잉크'의 높이를 계산하여 패널의 수직 정중앙에 텍스트를 배치합니다. 

### 3. 리눅스 Capability 활용
`sudo` 없이도 프로그램이 네트워크 카드에 직접 접근할 수 있도록 `cap_net_raw` 권한을 실행 파일에 부여합니다. 이는 `make setup` 또는 `setup_autostart.sh` 실행 시 자동으로 처리됩니다.

## 🚀 실행 및 관리

*   **컴파일**: `make`
*   **권한 설정**: `make setup`
*   **실행**: `./colorlight_led` (또는 `sudo ./colorlight_led`)
*   **자동 실행 등록**: `./setup_autostart.sh`
